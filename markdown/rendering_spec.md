# レンダリング仕様およびWebXR（VR）実装の留意点 (Babylon.js版)

このドキュメントでは、本アプリケーションの Babylon.js 移行後の描画システムにおける設定と、WebXR VRモード動作時の制約・留意点についてまとめます。

---

## 1. 基本的なレンダリング仕様

### レンダラー／エンジン設定
- **API**: Babylon.js (v7)
- **オプション**: `{ preserveDrawingBuffer: true, stencil: true }` を有効化し、描画バッファの維持とステンシルバッファを利用した特殊効果のレンダリングに対応。
- **色空間 / カラーハンドリング**:
  - Babylon.js は内部でリニアワークフローおよび sRGB 色空間への自動補正を処理し、高コントラストで意図通りの色彩表現を保証します。
- **背景クリアカラー**:
  - デフォルトでは `#0b1118` に近いダークトーン (`new Color4(0.04, 0.07, 0.09, 1.0)`) を設定。背景モードの切り替えにより単色背景、または `GridMaterial` を適用した無限グリッド床との排他表示が可能です。

### シャドウ（影）システム
- **シャドウマップ**: `ShadowGenerator` を使用。
- **フィルタリング**: `useBlurExponentialShadowMap = true`（ブラー指数シャドウマップ）および `useKernelBlur = true`、`blurKernel = 32` を適用。これにより、ピクセル感のない滑らかで自然にボケた影を描画します。
- **解像度**: UIの設定値に応じて `256`〜`1024` に動的にリサイズ（デフォルト `1024`）。
- **光源設定**: 影制作用の平行光源 (`DirectionalLight`) の出力を調整し、影を生成。モデル（メッシュ）のロード時に、モデル全体が影のキャスターとしてジェネレーターに自動登録されます。

### 物理演算 (Havok)
- **エンジン**: `@babylonjs/havok` (WebAssembly版) を採用。
- **重力設定**: `scene.enablePhysics` を用い、MMD内の長さスケール補正（約12.5倍）を掛けたベクトルを物理エンジンに設定。UIから変更された場合も即座に同期されます。
- **MMD物理**: `babylon-mmd` に内包された CCD-IK および剛体・ジョイント連携（Ammo.js / Bulletに代わるHavokプラグイン経由でのバインド）を使用し、滑らかで安定した服や髪の揺れを実現しています。

## 2. WebXR (VR) モードにおける実装と重要な留意点

VRゴーグル（特にMeta QuestシリーズなどのモバイルVR）内でのステレオレンダリング時、特有の描画干渉やアーティファクトが発生します。Babylon.jsの標準的なWebXR機能（`WebXRDefaultExperience`）をベースに、以下の制御が行われています。

### ① 固定フォビエートレンダリング (FFR) の回避と最適化
- **現象**: 視野の中心以外の解像度を落とす FFR 機能により、グリッド床の細い線がかすれ、モアレ（ジャギー）や四角いもやのように見える現象が発生します。
- **対応**: Babylon.js では `WebXRDefaultExperience` がデバイスに最適な解像度バランスを自動調停します。必要に応じてセッション開始時に明示的にレンダラーのビューポート比率を均一に保ち、不連続なスペキュラやジャギーを抑制します。

### ② パススルー（背景透過）とコンポジタの合成
- **現象**: Questなどで現実の部屋の風景を背景にキャラクターを表示させたい場合、シーン全体の透過と、デバイス側のカメラ映像の合成処理が必要になります。
- **対応**:
  - `XrManager` にて、`WebXRFeatureName.XR_LEGACY_PASSTHROUGH` (パススルー機能) をトグル操作で有効化できるように構成。
  - パススルー有効時は、背景のクリアカラーおよび床メッシュのレンダリング順序や透明度を調整し、OS（WebXRコンポジタ）レベルでのカメラ透過合成を阻害しないようにします。

### ③ 二重ループ（描画競合）の防止（自動解消）
- **以前のバグ (Three.js)**: 通常カメラの `renderer.render()` と VRステレオカメラのループが同時に走り、WebGLのバッファやビューポートが破壊されるバグがありました。
- **Babylon.jsでの動作**:
  - Babylon.js では、XRモードに入ると `Engine` 自体が内包する描画ループ（`engine.runRenderLoop` 内で `scene.render()` を呼び出す）がXR用のステレオレンダリングに自動的・シームレスに切り替わります。
  - 通常カメラとVRカメラの二重ループ競合が発生しない構造になっているため、描画のバグやクラッシュが本質的に回避されています。

## 3. WebXR UI (HTML UIの重ね合わせ) に関する留意点

- **DOM Overlay (Quest等でのHTMLUI表示)**:
  - VR内でWebページ上のボタンなどを操作する場合、`WebXRDomOverlay` フィーチャーを有効化します。
  - **CSSフィルターのバグ回避**: Three.js版同様、VRへの投影時には `backdrop-filter: blur(...)` などのぼかしフィルターが「黒いもや」になるバグを引き起こす可能性があるため、CSS側のメディアクエリやクラス切り替えでぼかしを無効化し、不透明度を上げる等のフォールバック対応を推奨します。
  - **位置固定**: `position: fixed` を用いて、ビューポートに対する絶対位置（画面端など）を明示的に指定することで、VR内のコントローラー操作面でも正しくUI要素が固定・表示されるようになります。
