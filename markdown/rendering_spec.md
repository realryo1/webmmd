# レンダリング仕様およびWebXR（VR）実装の留意点

このドキュメントでは、本アプリケーションの描画システムにおける設定と、WebXR VRモード動作時の制約・留意点についてまとめます。

---

## 1. 基本的なレンダリング仕様

### レンダラー設定
- **API**: Three.js WebGLRenderer (r160)
- **オプション**: `{ antialias: true }` を有効化し、シャギーやジャギーの発生を抑制。
- **カラースペース**: `renderer.outputColorSpace = 'srgb'`
- **トーンマッピング**: `renderer.toneMapping = 4` (ACESFilmicToneMapping)
  - 水色などの強い色合いの彩度を維持したまま白飛びを防止。
  - 露出値 (`renderer.toneMappingExposure`) は `0.8` に設定。

### シャドウ（影）システム
- **シャドウマップタイプ**: `PCFSoftShadowMap` (2) を採用し、境界が柔らかい影を描画。
- **解像度**: 設定（`shadowMapSize`）に応じて `256`〜`1024` に動的にリサイズ（デフォルト `512`）。
- **光源**: DirectionalLight に対し `castShadow = true` に設定し、影を生成。

## 2. WebXR (VR) モードにおける実装と重要な留意点

VRゴーグル（特にMeta QuestシリーズなどのモバイルVR）内でのステレオレンダリング時、特有の描画干渉やアーティファクトが発生します。これらを防ぐために以下の実装が行われています。

### ① 固定フォビエートレンダリング (FFR) の無効化 [超重要]
- **現象**: FFRが有効であると、視野の中心だけを高解像度で描き、周辺の解像度を極端に落とします。この結果、座標平面（グリッド）のような極細の線が解像度の境界でかすれてジャギー（モアレ）となり、「画面中央に四角い半透明のもやがある」ように見えてしまいます。また、反射光（スペキュラ）も境界で不連続になる現象を引き起こします。
- **対応**: VRセッション開始時に `renderer.xr.setFoveation(0)` を明示的に呼び出し、フォビエートレンダリングを完全に無効化します。これにより全画面が均一にフル解像度でレンダリングされ、境界アーティファクトが消滅します。

### ② シーン背景の透過処理とコンポジタ干渉防止
- **現象**: `scene.background` に色（Colorオブジェクト）が指定されたままだと、WebXRのコンポジタ（OSの合成システム）が左右の目のフレームバッファを合成する際、アルファ値と干渉してフレームバッファの境界が「半透明の黒いもや」として焼き付くバグが発生します。
- **対応**: 
  - VRセッション開始時 (`_onSessionStart`): `_scene.background = null` に退避させます。
  - VRセッション終了時 (`_onSessionEnd`): 元の背景色を復元します。

### ③ 重複レンダリング（二重ループ）の完全ガード
- **現象**: 通常画面用の描画ループ（`pu.render` 内の `renderer.render`）と、WebXRManager が駆動する `setAnimationLoop(_xrRenderLoop)` が同時に走ると、1つのWebGLコンテキストに対して通常のカメラとVRのステレオカメラが交互に描画命令を送り、描画バッファやビューポートが破壊されます。
- **対応**: 通常の `render()` 関数の冒頭に以下のガードを入れ、VR起動中は通常の描画を完全にスキップします。
  ```javascript
  if (this.renderer.xr.enabled && this.renderer.xr.isPresenting) {
    return;
  }
  ```

### ④ VRセッション中のシャドウ（影）の一時無効化
- **現象**: WebXRによる左右の目の描画中にシャドウマップの自動更新が有効なままだと、深度バッファの不整合や、描画パフォーマンスの急激な低下、またはWebGLエラーを引き起こす原因になります。
- **対応**: 
  - VRセッション開始時: `renderer.shadowMap.autoUpdate = false` にし、各メッシュおよびライトの `castShadow` を一時的に `false` に切り替えます。
  - VRセッション終了時: 通常モードの影設定をそのまま復元します。

## 3. DOM Overlay (HTMLUIの重ね合わせ) に関する留意点

- **backdrop-filterの不具合**:
  - `backdrop-filter: blur(...)` を適用したHTML要素（ボタンなど）を `dom-overlay` 経由でVRに投影すると、3D空間上の境界線がぼかされ、画面中央に「ぼやけた半透明の黒いもや」として投影されるバグが発生します。
  - **対策**: VRセッション適用中はCSSで `backdrop-filter: none !important` に強制変更し、代わりに背景の不透明度を上げることで視認性を維持します。
- **要素の配置ずれ**:
  - VR空間への投影時、絶対位置（`position: absolute`）や `inset` を使用すると配置が画面中央付近にズレるバグがあります。
  - **対策**: `position: fixed` を使用し、個別の `bottom`, `left` などの位置プロパティを明示的に指定することで、VR空間でも左下に安定して固定されるようにします。
