# モーション正常非反映問題に関する調査履歴と引き継ぎ資料

特定のモデル（`1mikunude.pmx`）において、モーションの「お尻を落とす上下動（しゃがみ動作）」が反映されない不具合について、これまでの調査結果とデバッグ用に追加したコンソールログの履歴、および次のステップへの引き継ぎ事項をまとめました。

---

## 1. 発生している不具合と現状
- **現象**: 
  - 改変前の正常なモデル（`yyb hatsune miku_10th_v1.02.pmx`）では正常に腰（センター）が上下動してしゃがむが、改変後モデル（`1mikunude.pmx`）では腰の位置が上下動せず、その場に留まる（ただし回転等の他のモーションは適用されている）。
- **デバッグによる測定値**:
  - `Time: 37s` 付近（しゃがんでいる状態）で一時停止してボーン座標をダンプした結果：
    - **正常モデル**: `[Bone: センター] Live Local Pos Y: -1.018`（アニメーションの指示値に従ってマイナスに移動している）
    - **おかしいモデル**: `[Bone: センター] Live Local Pos Y: 0.000`（アニメーション指示値は正常モデルと同じく `-1.05` などが入っているが、ローカル位置が `{0, 0, 0}` の初期位置から全く動いていない）

---

## 2. 調査履歴とこれまでに判明した事実
1. **つま先IKの補正**:
   - `1mikunude.pmx` で「つま先IK」の変形階層（`transformOrder`）が足IKと同じ `1` になっていた不整合を検知。モデルロード時に自動的に `足IK + 1`（即ち `2`）に補正するロジックを `MmdManager.js` の `loadModel` 内に追加したが、これ単体では上下動は解決しなかった。
2. **物理演算による上書き拘束の疑いと検証**:
   - `1mikunude.pmx` は素体モデルのため、体幹や髪などの剛体数が正常モデルの倍以上存在し、最適化漏れによって「物理（Physics）」タイプの剛体がセンターの移動を `{0,0,0}` に強力にロックしている疑いがあった。
   - 対策として、シーン全体の物理エンジン（`PhysicsEngine`）のタイムステップ（`TimeStep`）を強制的に `0` にして物理更新を完全にフリーズさせる無効化ロジックを `setPhysicsDisableGlobally` に追加した。
   - **結果**: 物理を完全に無効化し、髪やしっぽが完全に静止する状態を作っても、おかしいモデルのセンターは `{0,0,0}` のまま変化しなかった。これにより、**物理演算が原因ではない**ことが確定。
3. **ボーンのインスタンス型とフラグの検証**:
   - 移動アニメーションが適用されない原因として、ボーンが移動可能ボーン（`MmdRuntimeMovableBone`）として初期化されていない疑いがあり、ボーンのコンストラクタと非公開プロパティをダンプした。
   - **結果**: 正常モデル・おかしいモデルともに、ボーンのコンストラクタは同じ `_MmdRuntimeBone`（`MmdRuntimeBone` の内部クラス）であり、フラグも共に `flag: 30`（移動可能フラグ `0x0004` を含む）で完全に一致していた。
   - したがって、ボーン自体の設定やクラス型にも差異はない。

---

## 3. 追加したコンソールデバッグ処理（MmdManager.js）
スペースキー押下時に、リアルタイムな状態を詳細に出力する `dumpCurrentPoseAndMotion()` を `MmdManager.js` の末尾に追加しています。

現在、以下の情報をダンプします：
- **`MmdRuntime` / `MmdPhysics`** の生のJavaScriptオブジェクト（物理が本当に `enabled: false` になっているかの確認用）
- **`MmdModel`** の生のオブジェクト（バインド状態の確認用）
- **主要ボーン（全ての親、センター、グルーブ、腰、下半身、左右足IK）** の情報：
  - ボーンのコンストラクタ名
  - ボーンオブジェクトの非公開プロパティ（`flag` や `transformAfterPhysics` の実体値）
  - 現在のローカル移動量（`Live Local Pos`）およびワールド座標（`Live World Pos`）
  - VMDモーションから読み込まれた、現在の再生フレームに最も近いキーフレームの座標指示値（`Closest Keyframe`）

---

## 4. 次のチャットへの引き継ぎ事項（仮説とデバッグ方針）
ボーンの型もフラグも正常モデルと同一であるにもかかわらず位置移動が適用されないため、**「アニメーションをモデルにバインドする際のマッピング処理において、1mikunude.pmx のセンターボーンが移動トラック（Position）に結びつけられていない（バインド漏れしている）」** 可能性が極めて高いです。

### 次に調査すべきこと
1. **`movableBoneBindIndexMap` の中身のダンプと分析**:
   - `MmdModel` がバインドしている `_currentAnimation` の内部プロパティに、`boneBindIndexMap`（回転のみボーンのマップ）と `movableBoneBindIndexMap`（移動＋回転ボーンのマップ）があります。
   - 正常モデルとおかしいモデルで、この `movableBoneBindIndexMap` の生の数値配列（特に「センター」のインデックス `2` がバインドされている位置）を比較し、おかしいモデルで登録漏れ（`-1` や `undefined` になっている）が発生していないか調査してください。
2. **モデルロード時のインデックス構造の差**:
   - `1mikunude.pmx` は `全ての親` が Index `0`、`yyb hatsune miku` は `全ての親` が Index `1` となっており、インデックスの開始位置にずれがあります。このインデックスのずれが、アニメーションバインド時の配列走査においてバグを引き起こしていないか確認してください。
